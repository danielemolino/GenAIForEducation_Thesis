# Setup iniziale (prima volta)

## 0) Prerequisiti2 e install dipendenze
```bash
# Node/Yarn (monorepo OHIF + lerna)
node -v
yarn -v
cd /mnt/d/GenAIforEdu/Viewer
yarn config set workspaces-experimental true
yarn install

# Python backend
cd /mnt/d/GenAIforEdu
python -m venv .venv
source .venv/bin/activate   # su PowerShell: .venv\Scripts\Activate.ps1
pip install --upgrade pip setuptools wheel
pip install numpy pydicom fastapi nibabel pillow tifffile uvicorn
```

# Avvio rapido - GENEDU (VM web)

Tre terminali separati su `genedu`:

## 1) Orthanc (Docker)
```bash
cd /mnt/d/GenAIforEdu/Viewer
yarn orthanc:up
```

## 2) Viewer (web app)
```bash
cd /mnt/d/GenAIforEdu/Viewer
yarn dev:orthanc
```

## 3) Backend orchestratore (chiama servizi remoti)
```bash
cd /mnt/d/GenAIforEdu/BackendModelli
pip install numpy pydicom fastapi nibabel pillow tifffile uvicorn
export BACKEND_MODE="full"
export GENERATION_ENGINE="real"
export REMOTE_INFERENCE_URL="http://slurmsrv:4450"
uvicorn main:app --host 0.0.0.0 --port 8000 --access-log
```

# Servizi modello su GN03 + gateway unico su 8001

Architettura:
- `genedu` chiama `http://slurmsrv:4450`
- `slurmsrv:4450 -> gn03:8001` (gateway)
- gateway inoltra localmente a:
  - CT worker `127.0.0.1:8101`
  - XRay worker `127.0.0.1:8102`

## 1) CT worker (gn03:8101)
```bash
cd /path/to/GenAIForEducation_Thesis/BackendModelli
source .venv/bin/activate
uvicorn remote_services.workers.ct_worker:app --host 0.0.0.0 --port 8101 --access-log
```

## 2) XRay worker (gn03:8102)
```bash
cd /path/to/GenAIForEducation_Thesis/BackendModelli
source .venv/bin/activate
export XGEM_API_ROOT="/path/to/XGeM_API"
uvicorn remote_services.workers.xray_worker:app --host 0.0.0.0 --port 8102 --access-log
```

## 3) Gateway unico (gn03:8001)
```bash
cd /path/to/GenAIForEducation_Thesis/BackendModelli
source .venv/bin/activate
export CT_WORKER_URL="http://127.0.0.1:8101"
export XRAY_WORKER_URL="http://127.0.0.1:8102"
uvicorn remote_services.gateway.main:app --host 0.0.0.0 --port 8001 --access-log
```
